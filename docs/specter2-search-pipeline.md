# Text-to-Candidates Search Pipeline (SPECTER2 + remax)

End-to-end example: take a natural-language query, retrieve 100 candidate
papers from a corpus of Semantic Scholar SPECTER2 embeddings using remax
as the Stage 1 binary filter.

## Prerequisites

```bash
pip install remax transformers torch numpy
```

## 1. Load Pre-computed Embeddings

Semantic Scholar publishes SPECTER2 embeddings for 100M+ papers as part
of the [S2AG dataset releases](https://www.semanticscholar.org/product/api/tutorial).
The `embeddings` dataset ships as gzipped JSON-lines files:

```json
{"corpusid": 204838007, "vector": [0.123, -0.456, ...]}
```

If you have these staged in S3 (or any local path), load them into a
single NumPy array:

```python
import gzip
import json
import numpy as np
from pathlib import Path

def load_s2_embeddings(glob_pattern: str) -> tuple[np.ndarray, list[str]]:
    """Load S2 SPECTER2 embeddings from gzipped JSON-lines files.

    Returns (vectors, corpus_ids) aligned by index.
    """
    vectors, ids = [], []
    for path in sorted(Path(".").glob(glob_pattern)):
        opener = gzip.open if path.suffix == ".gz" else open
        with opener(path, "rt") as f:
            for line in f:
                rec = json.loads(line)
                if rec.get("vector"):
                    vectors.append(rec["vector"])
                    ids.append(str(rec["corpusid"]))
    return np.array(vectors, dtype=np.float32), ids

corpus_vectors, corpus_ids = load_s2_embeddings(
    "s2-embeddings/*.jsonl.gz"
)
# corpus_vectors: (n, 768) float32
# corpus_ids:     ["204838007", "73346332", ...]
```

## 2. Build the Corpus (offline, once)

```python
from remax import Corpus

# Build the index. center=True subtracts the corpus mean before
# sign-packing AND persists the mean as mean.npy so that search()
# can auto-center queries. This is load-bearing for retrieval
# quality. (Raw SPECTER2: R@10 ≈ 0.47; centered: R@10 ≈ 0.64
# on the v0.1.0 bench.)
corpus = Corpus.build(
    "my_index",
    vectors=corpus_vectors,
    ids=corpus_ids,
    seed=42,
    center=True,
)
# Writes:
#   my_index/index.bin  — 32-byte header + packed sign-bit codes
#   my_index/meta.db    — SQLite: rowid → record_id + JSON meta
#   my_index/mean.npy   — corpus mean vector (only when center=True)
```

On-disk footprint: 768 / 8 = **96 bytes per vector** plus the
32-byte header. A 1M-paper corpus is ~92 MB of codes.

## 3. Encode Queries and Search

The corpus embeddings were generated by S2 using the SPECTER2
**proximity** adapter (document↔document similarity). For ad-hoc
search queries, use the **`adhoc_query`** adapter — it's trained
for the asymmetric query→document retrieval setting:

```python
import torch
from transformers import AutoTokenizer, AutoModel

# --- Load SPECTER2 with the query adapter ---
tokenizer = AutoTokenizer.from_pretrained("allenai/specter2_base")
query_model = AutoModel.from_pretrained("allenai/specter2_base")
query_model.load_adapter("allenai/specter2_adhoc_query", source="hf", set_active=True)
query_model.eval()


def encode_query(text: str) -> np.ndarray:
    """Encode a search query with the SPECTER2 adhoc_query adapter."""
    inputs = tokenizer(
        text,
        padding=True,
        truncation=True,
        max_length=512,
        return_tensors="pt",
    )
    with torch.no_grad():
        out = query_model(**inputs)
    mask = inputs["attention_mask"].unsqueeze(-1).float()
    emb = (out.last_hidden_state * mask).sum(1) / mask.sum(1)
    return emb[0].cpu().numpy().astype(np.float32)
```

> **Why `adhoc_query` and not `proximity`?** SPECTER2 ships separate
> adapters tuned for different tasks. The proximity adapter is symmetric
> (document↔document); `adhoc_query` is asymmetric (short query →
> document), trained on click-through and relevance data. Both produce
> 768-d vectors in the same embedding space. Use `adhoc_query` when your
> input is a natural-language question or keyword phrase; use `proximity`
> when comparing two documents.

Now search:

```python
from remax import Corpus

# Load (once per process)
corpus = Corpus("my_index")

query_text = "efficient methods for training large language models"
query_vec = encode_query(query_text)

# Corpus auto-centers the query using the stored mean.
# No manual centering step.
results = corpus.search(query_vec, k=100)

for r in results[:5]:
    print(f"  #{r.rank}  d={r.distance}  {r.record_id}")
```

Output (illustrative):
```
  #0  d=287  204838007
  #1  d=291  261100919
  #2  d=294  249063163
  #3  d=296  215416146
  #4  d=298  259950998
```

Distances are Hamming (number of bit disagreements out of 768).
Lower = more similar. Theoretical range: 0 to 768. The `record_id`
is the S2 Corpus ID — resolve to metadata via the
[S2 Paper API](https://api.semanticscholar.org/api-docs/graph#tag/Paper-Data/operation/get_graph_get_paper).

## 4. Stage 2 Rerank (optional)

The 100 candidates from remax are coarse. For a production pipeline,
rerank them with full-precision scores — either via remex or a
cross-encoder:

```python
# Option A: remex asymmetric inner-product rerank
from remex import Quantizer

# Re-encode the candidate subset at higher precision
candidate_indices = [int(r.record_id) for r in results]
candidate_vecs = corpus_vectors[...]  # look up by corpus ID
q = Quantizer(d=768, bits=4, seed=42)
compressed = q.encode(candidate_vecs)
final_idx, scores = q.search(compressed, query_vec, k=10)

# Option B: cross-encoder rerank (highest quality, slowest)
from sentence_transformers import CrossEncoder

reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")
pairs = [(query_text, get_title_abstract(cid)) for cid in candidate_indices]
scores = reranker.predict(pairs)
top10 = np.argsort(-scores)[:10]
```

## Notes

**Centering matters.** SPECTER2 embeddings have non-zero per-dimension
means (one dimension has mean ≈ 15.5). Charikar SimHash assumes
mean-zero inputs. Centering closes the gap between naive sign-bit
encoding and Lloyd-Max 1-bit quantization, which adaptively places
its boundary at the per-dimension mean.

**Centering is automatic.** `Corpus.build(center=True)` persists the
corpus mean as `mean.npy`. On load, `Corpus` detects the mean file
and `search()` subtracts it from queries before encoding. Callers
pass raw encoder output — no manual centering step. The mean is
accessible via `corpus.mean` if needed externally (e.g., for
interop with the low-level `SignBitQuantizer` API).

**Adapters matter for queries.** The S2 corpus was embedded with the
proximity adapter. For search-style queries (short text → relevant
documents), use `allenai/specter2_adhoc_query`. For document-to-document
similarity (e.g., "find papers like this one"), use
`allenai/specter2` (proximity). Both adapters produce compatible
768-d vectors. A refreshed variant (`allenai/specter2_aug2023refresh_adhoc_query`)
is also available with slightly newer training data.

**Scaling.** At 200M vectors (the full S2 corpus), the index is
~18.3 GB of packed codes. A brute-force Hamming scan is feasible on
a single machine with NumPy. For sub-linear access, pair with
remex's `IVFCoarseIndex` for cell routing (not yet integrated;
tracked in the remax roadmap).
